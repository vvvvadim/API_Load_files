# Система загрузки файлов API

Современное веб-приложение для загрузки и управления файлами с акцентом на фотографии и управление заказами.

## Возможности

- **Современный пользовательский интерфейс**
  - Адаптивный дизайн для работы на компьютерах и мобильных устройствах
  - Чистый и интуитивно понятный интерфейс загрузки файлов
  - Мгновенная обратная связь при выборе файлов
  - Стилизованные кнопки и элементы формы

- **Управление файлами**
  - Поддержка загрузки нескольких файлов
  - Проверка форматов изображений
  - Отображение статуса выбора файлов в реальном времени
  - Предварительный просмотр списка файлов перед загрузкой

- **Управление заказами**
  - Отслеживание номеров заказов
  - Организованное хранение файлов по номерам заказов
  - Безопасная обработка файлов

- **Краткое описание работы программы**

Алгоритм работы следующий:
1) создается заказ наряд загружаются фото,
2) 1С принимает заказ наряд методом GET в /api/order
3) Ставим статусы метод POST /api/order/{tag}
структура POST:
{
  "tag": "string",
  "status_media": "string",
  "status_order": "string"
}
	а)Статус NEW ставиться автоматом для всех заказ нарядов и медиафайлов
	б)Статус ACCEPTED для опции status_media проставляет статус для медиа файлов как принятые, 
      статус заказ наряда не меняется, в п.2 при выдаче этого заказ наряда будут отображаться только те медиа у 
      которых статус NEW. Это нужно для дозагрузки файлов.
	в)Статус DELETED для опции status_media проставляет статус для медиа файлов как удаленные. Собственно удаляет файлы.
	г)Статус CLOSED для опции status_order закрывает заказ наряд. При этом автоматом проставляется статус ACCEPTED для 
      опции status_media. Закзаз наряд закрывается, изменить его нельзя, но при обращении он покажет галерею с 
      фотоматериалом данного заказ наряда
	д)Статус DELETED для опции status_order удаляет заказ наряд. При этом автоматом проставляется статус DELETED для 
      опции status_media. Заказ наряд помечается как удаленный, при попытке обращения к нему галерея не доступна 
      выходит страничка с информацией что данный заказ наряд в загрузке фотоматериала не нуждается.

## Технический стек

- **Серверная часть**
  - Сервер на Python
  - Фреймворк FastAPI
  - Надежная обработка ошибок
  - Эффективная обработка файлов

- **Клиентская часть**
  - HTML5
  - CSS3 с современными возможностями
  - JavaScript для динамических взаимодействий
  - Адаптивный дизайн с медиа-запросами
  - Интеграция пользовательского шрифта (Open Sans Condensed)

## Требования

- Docker
- Docker Compose

## Установка и запуск

1. Клонируйте репозиторий:
```bash
git clone [адрес-репозитория]
```

2. Перейдите в директорию проекта:
```bash
cd [название-директории]
```

3. Запустите приложение с помощью Docker Compose:
```bash
docker-compose build
docker-compose up -d
```

Приложение будет доступно по адресу: http://localhost:8000

## Остановка приложения

Для остановки приложения выполните:
```bash
docker-compose down
```

## Структура проекта

Проект организован следующим образом:

```
app/
├── main.py          # Основная точка входа приложения
├── media.py         # Функциональность обработки медиафайлов
├── orders.py        # Система управления заказами
└── ui/
    └── index1.html  # Основной пользовательский интерфейс
```

## Использование

1. Откройте веб-интерфейс в браузере по адресу http://localhost:8000
2. Введите или выберите номер заказа
3. Нажмите "Обзор" для выбора файлов
4. Выберите одно или несколько изображений
5. Нажмите "Отправить" для загрузки файлов
